# Участие в разработке (Contributing)

Спасибо за интерес к проекту __SubFlux__!

Этот документ описывает, как корректно взаимодействовать с проектом:
сообщать об ошибках, предлагать улучшения и вносить изменения в код.

---

## Статус проекта
Проект находится в активной разработке.
Архитектура, API и схема базы данных могут меняться без обратной совместимости.

---

## Issues (ошибки и предложения)
GitHub Issues используются для:
- Сообщений об ошибках
- Предложений новой функциональности
- Обсуждения улучшений

Перед созданием Issue:
- Проверьте, нет ли уже похожей проблемы в существующих Issue
- Убедитесь, что проблема воспроизводится на актуальной версии проекта

### Сообщение об ошибке (Bug report)
Создайте [Issues](https://github.com/NikPACodes/SubFlux/issues) типа __"Сообщение об ошибке"__ и укажите:
- Что ожидалось и что произошло фактически
- Шаги для воспроизведения
- Окружение (ОС, версия Python, Django, способ запуска)

### Запрос функциональности (Feature request)
Создайте Issue типа __"Запрос функциональности"__ и укажите:
- Какую проблему решает предложение
- Желаемое поведение/результат
- (Опционально) Идеи реализации

⚠️ Важно: вопросы безопасности __не публикуйте в Issues__ — используйте [SECURITY](./SECURITY.md).

---

## Pull Requests (внесение изменений в код)

### Общий порядок
1. Сделайте [fork](https://github.com/NikPACodes/SubFlux/fork) репозитория
2. Создайте отдельную ветку:
   - `feat/<краткое-описание>` — для новой функциональности
   - `fix/<краткое-описание>` — для исправлений
   - `docs/<краткое-описание>` — документация
3. Внесите изменения
4. Убедитесь, что тесты проходят
5. Создайте Pull Request в основную ветку

### Требования к Pull Request
- PR должен решать __одну задачу__
- Изменения должны быть понятны без чтения кода целиком
- При изменении бизнес-логики желательно добавлять тесты
- Не добавляйте секреты и пароли в репозиторий

Перед отправкой PR используйте чек-лист из шаблона Pull Request.

## Локальный запуск (кратко)

Общие шаги для локальной разработки:
- Скопируйте `.env.example` в `.env`
- Установите зависимости
- Примените миграции
- Запустите сервер
