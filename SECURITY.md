# Политика безопасности

## Сообщения об уязвимостях
Пожалуйста, __НЕ создавайте публичные GitHub Issue__ для сообщений об уязвимостях.

Если вы нашли проблему безопасности:
- напишите владельцу репозитория напрямую через GitHub (private message),
- либо используйте контактные данные, указанные в профиле владельца.

## Что указать в сообщении
- описание уязвимости;
- шаги для воспроизведения;
- затронутые компоненты (модели, API, сервисы);
- потенциальный риск;
- Предложения по исправлению (если имеются)

Мы постараемся ответить как можно быстрее и сохранить информацию конфиденциальной до выхода исправления.
